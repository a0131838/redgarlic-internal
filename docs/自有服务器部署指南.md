# 红蒜系统自有服务器部署指南

本项目按 `tuition-scheduler` 的发布纪律搭建，但应用、数据库、端口、域名全部独立。

## 1. 准备
- Ubuntu 22.04 或 24.04 服务器
- 域名 `redgarlicai.com` 已解析到服务器公网 IP
- 独立 Neon PostgreSQL 连接串

## 2. 填配置
```bash
cp ops/server/.deploy.env.example ops/server/.deploy.env
```

至少填写：
- `DOMAIN`
- `LETSENCRYPT_EMAIL`
- `NEXT_PUBLIC_APP_URL`
- `DATABASE_URL`
- `DIRECT_DATABASE_URL`
- 如果接 COS，再填写：
  - `SHARED_FILE_STORAGE_DRIVER=s3`
  - `SHARED_FILE_S3_BUCKET`
  - `SHARED_FILE_S3_REGION`
  - `SHARED_FILE_S3_ENDPOINT`
  - `SHARED_FILE_S3_ACCESS_KEY_ID`
  - `SHARED_FILE_S3_SECRET_ACCESS_KEY`

高风险提醒：
- 生产数据库固定为 Neon，不要写 localhost
- 发布前先执行 `bash ops/server/scripts/quick_check.sh`
- 生产发布不要跳过 `verify_release_docs.sh`

## 3. 服务器初始化
```bash
sudo bash ops/server/scripts/bootstrap_server.sh ops/server/.deploy.env
sudo -u deploy bash ops/server/scripts/deploy_app.sh ops/server/.deploy.env
sudo bash ops/server/scripts/configure_nginx.sh ops/server/.deploy.env
sudo bash ops/server/scripts/setup_ssl.sh ops/server/.deploy.env
sudo bash ops/server/scripts/setup_backup_cron.sh
```

## 4. 验收
- 打开 `https://redgarlicai.com/`
- 打开 `https://redgarlicai.com/admin/login`
- 打开 `https://redgarlicai.com/api/health`
- 上传一份共享文件，确认可预览、可下载

## 5. 腾讯云 COS 配置

推荐在正式环境直接使用 COS，而不是继续长期依赖本地磁盘。

`.deploy.env` 中这样填：

```env
SHARED_FILE_STORAGE_DRIVER=s3
SHARED_FILE_S3_BUCKET=your-bucket-1250000000
SHARED_FILE_S3_REGION=ap-singapore
SHARED_FILE_S3_ENDPOINT=https://cos.ap-singapore.myqcloud.com
SHARED_FILE_S3_FORCE_PATH_STYLE=false
SHARED_FILE_S3_ACCESS_KEY_ID=AKIDxxxxxxxxxxxxxxxx
SHARED_FILE_S3_SECRET_ACCESS_KEY=xxxxxxxxxxxxxxxx
```

说明：
- 如果 COS Bucket 在新加坡，通常 Region 用 `ap-singapore`
- Endpoint 一般形如 `https://cos.<region>.myqcloud.com`
- 当前代码按 S3 兼容方式接入，腾讯云 COS 可直接使用
- 切到 COS 后，新上传文件会进 COS；历史本地文件仍可继续读取

建议：
- 先在测试环境填 COS 参数验证上传、预览、下载
- 确认没问题后再在正式环境切换 `SHARED_FILE_STORAGE_DRIVER=s3`

## 6. 后续发布
```bash
sudo -u deploy bash ops/server/scripts/deploy_app.sh ops/server/.deploy.env
```
