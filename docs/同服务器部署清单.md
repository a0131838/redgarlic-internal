# 同服务器部署清单

红蒜系统和排课系统可以共用同一台服务器，但必须保持运行时隔离。

## 当前隔离基线
- 排课系统继续保留自己的目录、端口、Nginx 站点、PM2 进程、数据库和对象存储
- 红蒜系统使用：
  - `APP_DIR=/opt/redgarlic-internal`
  - `APP_PORT=3001`
  - `APP_NAME=redgarlic-internal`
  - `DOMAIN=redgarlicai.com`
  - 独立 Neon 数据库
  - 独立 COS Bucket `redgarlic-files-1403694756`

## 上线前确认
- `redgarlicai.com` 已解析到服务器 `43.128.46.115`
- 服务器 80/443 已放通
- 排课系统当前没有占用 `3001`
- 生产配置文件 `ops/server/.deploy.env` 已填好
- 仓库地址 `REPO_URL` 已替换为真实地址

## 服务器执行顺序
```bash
cd /opt/redgarlic-internal
```

首次部署按顺序：

```bash
sudo bash ops/server/scripts/bootstrap_server.sh ops/server/.deploy.env
sudo -u deploy bash ops/server/scripts/deploy_app.sh ops/server/.deploy.env
sudo bash ops/server/scripts/configure_nginx.sh ops/server/.deploy.env
sudo bash ops/server/scripts/setup_ssl.sh ops/server/.deploy.env
sudo bash ops/server/scripts/setup_backup_cron.sh
```

后续发布：

```bash
sudo -u deploy bash ops/server/scripts/deploy_app.sh ops/server/.deploy.env
```
